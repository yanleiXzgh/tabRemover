# TapRenamer重构版部署指南

## 准备工作

### **检查环境**
确保你有以下环境：
1. Chrome浏览器或Edge浏览器
2. 开发者模式权限

### **文件准备**
两种部署方式：

**方式一：使用ES6模块版本（适合开发者）**
```
tabRemover/
├── popup/
│   ├── popup.html
│   ├── popup.js
│   ├── style.css
├── content-scripts/
│   ├── content-script.js
├── background/
│   ├── background.js
├── src/           # ES6模块源码
├── manifest.json  # 已配置为ES6模块版本
├── images/        # 图标
```

**方式二：使用传统打包版本（适合用户）**
```
tabRemover/dist/
├── popup/
│   ├── popup.html
│   ├── style.css
├── background.js    # 打包后的后台脚本
├── content-script.js # 打包后的内容脚本
├── popup.js         # 打包后的popup脚本
├── manifest.json    # 传统版本manifest
├── images/          # 图标
```

## 部署步骤

### **步骤1：加载扩展**
1. 打开Chrome或Edge浏览器
2. 进入扩展程序页面：
   - Chrome: chrome://extensions/
   - Edge: edge://extensions/
3. 点击右上角的"开发者模式"按钮，启用开发者模式
4. 点击左上角的"加载已解压的扩展程序"按钮
5. 选择重构后的文件夹：
   - ES6模块版本：选择`tabRemover`文件夹
   - 传统版本：选择`tabRemover/dist`文件夹

### **步骤2：验证安装**
1. 检查扩展图标是否出现在浏览器工具栏
2. 点击扩展图标，确认弹出窗口正常显示
3. 扩展名称应为"标签页名修改"，版本3.0

## 功能测试

### **测试1：直接改名功能**
1. 打开任意网页（如`test.html`）
2. 点击扩展图标
3. 在输入框中输入新名称
4. 点击"修改名称"按钮
5. 观察标签页名称是否改变

### **测试2：正则表达式功能**
1. 点击扩展图标，切换到"正则表达式"标签
2. 输入规则名称（如"GitHub")
3. 输入网址模式（如"github.com"）
4. 点击"添加正则表达式"按钮
5. 切换到"主页"标签
6. 点击"快速修改"按钮
7. 访问github.com页面，名称应自动修改

### **测试3：开关设置功能**
1. 打开扩展弹出窗口
2. 找到"开启自动改名"开关
3. 切换开关状态
4. 查看chrome.storage.local中的设置变化

### **测试4：关键词添加功能**
1. 打开一个包含keywords和description meta标签的网页
2. 启用"添加关键词"开关
3. 刷新页面或使用自动改名功能
4. 检查标题是否包含关键词和描述

## 常见问题

### **Q1：扩展图标不显示**
- 确认扩展加载成功
- 检查manifest.json格式是否正确
- 重新加载扩展

### **Q2：功能不工作**
- 确认标签页不是特殊页面（chrome://, edge://）
- 检查权限设置
- 查看浏览器控制台日志

### **Q3：AI功能不工作**
- AIProcessor需要API密钥配置
- 需要在AIProcessor.js中设置API密钥
- 或者暂时禁用智能总结功能

### **Q4：页面切换不正常**
- 检查popup.html结构
- 确认CSS样式正常加载
- 查看JavaScript错误

3. 使用Chrome DevTools调试

### **查看日志**
1. 打开Chrome DevTools
2. 切换到Console面板
3. 查看扩展输出的日志信息

### **查看存储数据**
1. 打开Chrome DevTools
2. 切换到Application面板
3. 查看Local Storage中的数据

## 生产模式

### **打包发布**
1. 使用dist目录的传统版本
2. 确保manifest.json格式正确
3. 测试所有功能正常
4. 打包为crx文件发布

### **用户反馈**
- 收集用户使用反馈
- 根据反馈优化UI界面
- 改进用户体验

## 注意事项

### **安全性**
- API密钥需要妥善保管
- 不要公开API密钥
- 使用安全的配置管理

### **兼容性**
- Chrome浏览器测试通过
- Edge浏览器测试通过
- 可能需要调整manifest版本

### **性能**
- 重构后代码量增加，但性能影响较小
- 异步操作优化，用户体验提升
- 内存占用需监控

## 总结

TapRenamer重构版提供了两种部署方式：

**开发模式**：使用ES6模块版本，适合开发者修改和扩展
**生产模式**：使用传统打包版本，适合用户直接使用

重构后的项目具有以下优势：
1. 清晰的代码结构
2. 强大的错误处理机制
3. 良好的扩展性
4. 易于测试和维护

按照以上步骤部署和测试，确保重构后的扩展功能正常。
